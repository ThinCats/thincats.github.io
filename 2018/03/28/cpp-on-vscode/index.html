配置VS code的C/C++环境

Posted on 2018-03-28 | Edited on 2018-09-30 | In VS code

参考资料:

Visual Studio Code如何编写运行C、C++？ - 谭九鼎的回答 - 知乎 C/C++ for VS Code (Preview)

前言

今天去 ThoughtWorks 公司参观了一下, 看到了一些人在使用VS code进行前端的开发, 可能是受到它的颜值吸引, 想去试试, 所以准备配置VS code. 记录一下大致的流程以及遇到的问题.

流程

  1. 配置环境
  2. 安装VS Code 相关插件
  3. 配置工作区相应插件的配置文件
  4. 其他优化

详细操作

  1. 配置环境

VS code对Clang的支持较好, 所以这里将根据知乎的教程配置Clang的环境

  1. 到 LLVM 下载并安装Clang, 配置环境变量
  2. 到 MinGW 安装MinGW-w64 (MinGW似乎很久没维护, 而MinGW-w64正在维护)

  3. 合并 将安装好的MinGW根目录复制到Clang 中合并(即含bin, include文件夹), 合并是为了MinGW的头文件, 也可以不进行合并, 而在配置文件[^C配置文件]中修改

  4. 安装 VS Code相关插件

  • C/C++（就是有些教程里的cpptools）
  • C/C++ Clang Command Adapter：提供静态检测（Lint）
  • Code Runner
  • Include Autocomplete：提供头文件补全
  • C/C++ Snippets：Snippets 重用代码块
  1. 配置工作区相应插件的配置文件(四个json)

参考知乎教程:

  1. 初始化某文件夹为工作区 (以工作区为单位的配置文件对工作区内的环境有效)

  2. 在工作区下创建 .vscode文件夹 ( .vscode 文件夹将被VS code识别)

  3. 在.vscode文件夹新建以下配置文件 (或者生成)

    • 新建tasks.json (可通过命令寻找: Tasks: Configure...生成)

    • 新建 launch.json (用于调试)

    • c_cpp_properties.json(用于配置头文件等)

    • settings.json (用于个性化设置, 如Clang的格式控制, 自动保存等)

一些小问题

  1. 先开始我的 gdb无法使用, 调试exe时出现了以下信息

生成的exe确实可以运行的, 但无法调试, 后面找到原因:

clang的架构是x86_64, 而gdb是for i686的, 两版本不同.

解决即为重新安装x86_64的MinGW

  1. 在VS code需要使用intelliSense , 但ctrl + space无法使用

中文输入法快捷键冲突了, 按下 win + space切换到英文输入法, 并换键盘布局为美式键盘(英式键盘布局有点不同)
