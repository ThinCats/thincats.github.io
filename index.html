配置VS code的C/C++环境

Posted on 2018-03-28 | Edited on 2018-09-30 | In VS code

参考资料:

Visual Studio Code如何编写运行C、C++？ - 谭九鼎的回答 - 知乎 C/C++ for VS Code (Preview)

前言

今天去 ThoughtWorks 公司参观了一下, 看到了一些人在使用VS code进行前端的开发, 可能是受到它的颜值吸引, 想去试试, 所以准备配置VS code. 记录一下大致的流程以及遇到的问题.

流程

  1. 配置环境
  2. 安装VS Code 相关插件
  3. 配置工作区相应插件的配置文件
  4. 其他优化

详细操作

  1. 配置环境

VS code对Clang的支持较好, 所以这里将根据知乎的教程配置Clang的环境

  1. 到 LLVM 下载并安装Clang, 配置环境变量
  2. 到 MinGW 安装MinGW-w64 (MinGW似乎很久没维护, 而MinGW-w64正在维护)

  3. 合并 将安装好的MinGW根目录复制到Clang 中合并(即含bin, include文件夹), 合并是为了MinGW的头文件, 也可以不进行合并, 而在配置文件[^C配置文件]中修改

  4. 安装 VS Code相关插件

  • C/C++（就是有些教程里的cpptools）
  • C/C++ Clang Command Adapter：提供静态检测（Lint）
  • Code Runner
  • Include Autocomplete：提供头文件补全
  • C/C++ Snippets：Snippets 重用代码块
  1. 配置工作区相应插件的配置文件(四个json)

参考知乎教程:

  1. 初始化某文件夹为工作区 (以工作区为单位的配置文件对工作区内的环境有效)

  2. 在工作区下创建 .vscode文件夹 ( .vscode 文件夹将被VS code识别)

  3. 在.vscode文件夹新建以下配置文件 (或者生成)

    • 新建tasks.json (可通过命令寻找: Tasks: Configure...生成)

    • 新建 launch.json (用于调试)

    • c_cpp_properties.json(用于配置头文件等)

    • settings.json (用于个性化设置, 如Clang的格式控制, 自动保存等)

一些小问题

  1. 先开始我的 gdb无法使用, 调试exe时出现了以下信息

生成的exe确实可以运行的, 但无法调试, 后面找到原因:

clang的架构是x86_64, 而gdb是for i686的, 两版本不同.

解决即为重新安装x86_64的MinGW

  1. 在VS code需要使用intelliSense , 但ctrl + space无法使用

中文输入法快捷键冲突了, 按下 win + space切换到英文输入法, 并换键盘布局为美式键盘(英式键盘布局有点不同)

绝对收敛与条件收敛

1 收敛比较法

  1. 绝对收敛则原级数一定收敛
    • 证明采用柯西收敛准则
    • 内容:\(\displaystyle\sum{k=n+1}^{n+p}a_k < \displaystyle\sum_{k=n+1}^{n+p}|a_k| < \epsilon\)
    • 与比较判别法相比
      • 绝对收敛后成为同号级数，可以用比较判别法判定绝对值的收敛性
      • 比较判别法: 若\(a_n < b_n\), \(b_n\)收敛，那么\(a_n\)收敛

A_D判别法

Posted on 2018-03-19 | Edited on 2018-09-30 | In math

A-D 判别法(充分必要性)

  1. 适用 针对诸如\(\sum a_nb_n\)级数

1 基础变换: Abel变换(分部求和法)

  1. 内容: \(\displaystyle\sum^{n+p}_{k=n} a_nb_n = A_{n+p}b_{n+p} - A_{n-1}b_n - \sum^{n+p-1}_{k=n}A_k(b_{k-1} - b_k)\) (1.1)

    其中\(A_n\)为\(a_n\)部分和

  2. 类比: 分部积分法1
    • 内容: \(\int u(x)v'(x)dx = u(x)v(x) - \int v(x)u'(x)dx\)
    • 类比原则: 分部积分的离散版本
      \(define \ \Delta a_n = a_{n+1} - a_n, \ \sum b_n = B_n\)
      \(\displaystyle\sum^{n-1}B_i\Delta a_i = a_nB_n - \sum^{n}a_i\Delta B_{i-1}\) (1.2)

  3. 应用: Abel变换的(1.1)形式的左项为有限项的部分和，可以考虑在柯西收敛准则中使用

函数项级数

Posted on 2018-03-19 | Edited on 2018-09-30 | In math

函数项级数

定义:

无穷多个函数的'和' \(\sum u_n(x)\)

相关概念:

  1. 收敛点:
    对\(x_0 \in E,\) 使得 \(\sum u_n(x_0)\)收敛，则称\(x_0\)为收敛点1

  2. 收敛域: 函数项级数\(\sum u_n(x)\)的所有收敛点构成的集合, 记为 \(D\)

  3. 和函数: 由2所得定义 \(S(x) = \sum u_n(x), \ x \in D\)2

  4. 逐点收敛 由逐点定义所得到的和函数，则称\(\sum u_n(x)\) 在 \(D\)上 逐点收敛于\(S(x)\)

    • 为什么要在D上:
      • 因为不在D的x不为收敛点，退化的常数项级数不收敛.
    • 为什么收敛于一个函数:
      • 因为函数项的元素为函数，所以收敛于一个函数
    • 什么叫由逐点定义:
      • 可能因为这样的函数是一个一个离散x与对应的常数项级数所构成的，我猜是离散的原因。
  5. 部分和函数:
    • 定义: \(S_n(x), x \in D\)
    • 特征:
      • 依然基于区间\(D\)上
      • \(\lim\limits_{n \to \infty}S_n(x) = S(x)\)， 即部分和函数列[^部分和函数列]的敛散性与函数项级数相同

  1. 函数列退化为常数列↩

  2. 由收敛域及其收敛点所产生的常数项级数形成的集合构成的一个映射，即 \(f : D \to A, \ A = \{ \sum u_n(x_i)\}\)↩

函数列

Posted on 2018-03-19 | Edited on 2018-09-30 | In math

函数列

  1. 定义:元素为函数而非数的数列

函数列与函数级数的关系

  1. 任意给定一个函数列\(\{f_n(x)\}\), 对\(f_n(x)\), 可以通过构造\(u_n(x) = f_n(x) - f_{n-1}(x)\)来形成新的函数列,那么级数 \(\displaystyle\sum_n^\infty u_n(x)\) = 函数列元素\(f_n(x)\).
  2. 由此, 对级数收敛性的讨论可以转变成对函数列\(f_n(x)\)收敛性的讨论.
  3. 函数列与数列具有相似的部分,可以使用数列收敛性的讨论方法来进行.
  4. 同时函数具有的连续性也会带来其特殊的微分学性质(连续,可微,可积)

函数列的收敛意味着什么

  1. 类比数列, \(\{a_n\}\)收敛意味着\(\lim\limits_{n\to \infty} a_n = C\), 而针对函数列则是\(\lim\limits_{n\to\infty}f_n(x) = f(x)\)
  2. f(x)可以是多样化的, 如分段函数1, 而f(x)的定义域\(D\)取决于收敛域(参考 函数项级数.md)
  3. 把函数看成一种具有函数特征的一种特殊的数
